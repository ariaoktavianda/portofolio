  • Zero-Day

    Zero-day exploit adalah teknik serangan siber yang memanfaatkan kerentanan keamanan yang belum diketahui. Kerentanan ini dapat berada pada perangkat lunak, perangkat keras, atau firmware komputer. Ciri-ciri zero-day exploit: Tidak diketahui publik selama zero days Tidak memberikan kesempatan untuk dideteksi pada awalnya. Dapat menimbulkan masalah rumit sebelum ada yang menyadari ada yang salah. Sering kali menyebabkan kompromi pada sistem target. Contoh zero-day exploit: Serangan terhadap perangkat lunak Zoom yang memungkinkan penyerang mengakses perangkat pengguna. Serangan terhadap peramban web Google Chrome yang memungkinkan peretas memasang spyware dan malware akses jarak jauh di komputer korban. Serangan terhadap platform konferensi video populer yang memungkinkan peretas mengakses PC pengguna dari jarak jauh. Cara menanggulangi zero-day exploit: Membuat program yang memberi penghargaan kepada peneliti keamanan karena mengungkap kerentanan secara bertanggung jawab. Memberi tahu vendor perangkat lunak tentang kerentanan keamanan yang ditemukan.

  • Evil-twin (jagoan saya ini)

    Evil twin exploit adalah serangan siber yang memanfaatkan jaringan Wi-Fi palsu untuk mencuri data pribadi. Serangan ini juga dikenal sebagai serangan Man in the Middle (MitM). Cara kerja Penyerang membuat jaringan Wi-Fi palsu yang menyerupai jaringan asli Penyerang memposisikan diri di dekat pengguna akhir sehingga sinyalnya menjadi yang terkuat Pengguna tergoda oleh sinyal yang kuat dan terhubung ke jaringan palsu Data yang dibagikan oleh pengguna akan dikirim ke server yang dikendalikan oleh penyerang Tujuan serangan Mencuri kata sandi pengguna Mencuri data sensitif, seperti nomor kartu kredit dan kredensial login Menyusup lebih jauh ke perangkat penghubung Tempat yang sering terjadi Bandara, Kafe, Taman umum yang besar. Cara melindungi diri Waspada terhadap jaringan Wi-Fi yang tidak aman Tidak tergoda oleh sinyal yang kuat Tidak terhubung secara manual ke jaringan palsu Tidak memilih AP palsu secara otomatis.

  • Heartbleed

    Heartbleed adalah bug keamanan internet dalam pustaka perangkat lunak kriptografi OpenSSL. Bug ini memungkinkan eksploitasi protokol Transport Layer Security (TLS) dan Secure Sockets Layer (SSL) yang digunakan untuk mengamankan komunikasi internet untuk aplikasi web, email, pengiriman pesan, dan jaringan privat virtual (VPN) . Peretas menggunakan Heartbleed untuk mengelabui server SSL/TLS yang rentan agar memberikan informasi sensitif, termasuk nama pengguna dan kata sandi. "Heartbleed" adalah nama yang aneh untuk kesalahan komputer, bukan? Nah, nama ini berasal dari ekstensi detak jantung TLS, atau RFC6520, komponen OpenSSL yang bertujuan untuk memeriksa apakah koneksi antara dua perangkat aktif. Satu perangkat mengirimkan potongan data yang dikodekan ke perangkat lainnya, yang mengembalikan potongan data yang sama ke perangkat pertama, yang mengonfirmasi bahwa keduanya masih terhubung. Heartbleed adalah cacat dalam kode detak jantung, yang memungkinkan satu komputer mengambil data sensitif dari komputer lainnya selama pemeriksaan koneksi. Kode: Kode permintaan detak jantung: memcpy(bp, pl, payload). "memcpy()": menyalin data dari muatan permintaan detak jantung ke buffer respons. "bp": tujuan data yang disalin. "pl": lokasi sumber data. "payload": jumlah byte dari payload aktual yang diterima.

  • SQL Injection

    SQL injection adalah salah satu serangan berbahaya yang mengancam keamanan jaringan. Serangan ini memanfaatkan celah keamanan aplikasi web untuk menyuntikkan kode SQL berbahaya ke dalam database. Cara kerja SQL injection Penyerang memanfaatkan celah keamanan dalam input data Menyuntikkan perintah SQL berbahaya ke dalam query database Mengakses informasi yang tidak dimaksudkan untuk ditampilkan Melakukan berbagai hal, seperti menghapus dan mengubah data. Dampak SQL injection Kebocoran data, seperti informasi pelanggan, data diri, dan aset finansial Penyalahgunaan sistem Hilangnya berbagai data pada situs. Cara mencegah SQL injection Validasi input dan kueri berparameter Menggunakan parameterized statements Menggunakan escape characters Menggunakan firewall dan filter yang kuat Membersihkan semua input, tidak hanya input formulir web Melakukan validasi data dengan metode blacklisting dan whitelisting Menggunakan algoritma keamanan jaringan yang tepat.
